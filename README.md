# ForensiFlow  
## 面向 Android UI 自动化取证的智能体框架

---

## 📌 项目简介

ForensiFlow 是一个面向 Android 设备 UI 自动化取证的智能体框架。  
系统融合大语言模型（LLM）、视觉模块与混合解析策略，构建具备自主规划能力的移动端自动化取证体系，实现高效率、可扩展、符合法律规范的数字取证流程。

---

## 🚀 项目发展路线

---

### 🟢 第一阶段：基础智能体架构与视觉决策（已完成）

已完成核心智能体框架的搭建，实现：

- 基于大语言模型（LLM）的任务拆分与动作决策
- Vision 视觉模块对 Android UI 界面的识别与理解
- 自动生成操作步骤并驱动设备执行

本阶段验证了：

> 基于大模型的移动端 UI 自动化具备实际可行性。

---

### 🟡 第二阶段：混合解析引擎与效率优化（已完成）

构建“**XML 优先，视觉兜底**”的混合识别策略。

核心优化方向：

- 接入 Android UI Automator 底层 XML 节点解析
- 在 XML 解析失败或遇到复杂自定义控件时调用视觉模块
- 优化了上下文管理，同时更新了失败后执行逻辑
- 采用轻量级节点匹配算法完成大部分 UI 操作

预期效果：

- 显著降低大模型推理调用频率
- 提升整体执行效率
- 减少算力消耗与延迟

---

### 🔵 第三阶段：高级语义理解与上下文增强(规划中)

针对复杂 UI 场景中的识别误差与验证失败问题：

- 引入高级语义匹配算法
- 强化长流程上下文记忆机制
- 提升多步骤操作的状态保持能力

目标：

> 保证复杂取证流程中的稳定性与连续性。

---

### 🟣 第四阶段：知识驱动的动态任务规划

引入：

- 检索增强生成（RAG）
- 高级 Planner 任务规划模块
- 结构化提示词构建工程

系统将从“硬编码逻辑”转向“知识驱动决策”，实现：

- 基于取证任务库的动作流程检索
- 宏观任务自动拆解
- 微观路径动态规划
- 自适应执行策略生成

构建具备智能决策能力的自动化取证系统。

---

### 🟠 第五阶段：专业取证业务模块集成

针对不同应用场景构建专业化模块，例如：

- 微信取证模块
- 电报取证模块

核心能力包括：

- 差异化证据提取算法
- 面向目标应用的数据结构分析
- 自动化交互流程定制
- 可视化操作界面
- 实时取证进度与设备状态监控

实现业务级取证能力扩展。

---

### 🔴 第六阶段：司法合规与防篡改加固

- 完整日志记录（API 调用与设备交互过程）
- 哈希校验机制固化证据数据
- 防篡改与完整性验证机制
- 优化自动化操作对目标设备数据生态的影响

确保：

> 证据合法、原始、可追溯、可验证。

---

## 🔐 核心设计理念

- 无 Root 自动化取证
- 混合解析（符号 + 多模态）
- 知识驱动任务规划
- 模块化可扩展架构
- 符合法律合规要求
